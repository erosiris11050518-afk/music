# AI 语音配系统识别指引

这份文档用于后续继续扩展 AI 语音交互数据。当前语音入口只负责音响反推相关输入：音响数量、音响类型、型号、是否有源、并联串接、确认和撤销。功放、DSP、调音台、交换机、Dante 默认交给系统智能配接或专业页面处理，语音里听到这些设备时只提示用户继续说音响需求。

## 入口文件

- 识别归一化：`js/quick.js` 的 `normalizeSpeakerVoiceText`
- 意图解析：`js/quick.js` 的 `SP.parseSpeakerVoiceCommand`
- 反推应用：`js/quick.js` 的 `applySpeakerVoiceCommand`
- 回归测试：`tests/test-ui.js` 的 “AI语音配系统” 测试段

扩展规则时请同步增加测试，避免同音字修复影响已有说法。

## 处理流程

1. 先做文本归一化，把同音字、错字、口语量词统一成系统词。
2. 再按优先级解析意图：撤销 > 创建/追加/修改音响组 > 并联串接 > 确认 > 选型号 > 忽略系统设备 > 信息不全追问。
3. 创建音响组时输出槽位：`role`、`active`、`count`、`templateQuery`、`connectionMode`、`unitsPerChannel`。
4. 应用到反推卡片时，有源音响自动展开有源区域，无源音响继续参与功放反推。

语音连续识别时不要把浏览器本次返回的短片段直接当完整命令。`SpeechRecognition` 可能把一句话拆成多个 final result，所以全局语音入口会在每轮识别中重新组合完整 `event.results`，保留 final 前缀，用当前 interim 替换旧 interim，并在短暂停顿后提交未消费的文本。

## 归一化词库

| 口语或错字 | 系统词 |
| --- | --- |
| 全屏、全凭、全品、全平、全拼、全瓶、全频 | 全频 |
| 超底、超地、低音炮、重低音、低音 | 超低 |
| 有缘、有原、有圆、油源、优源、有电源、主动音响、自带功放、带功放音响 | 有源 |
| 无缘、无原、被动音响、不带功放 | 无源 |
| 支、之、枝、个、台、箱 | 只 |
| 八字超低、六字全频这类“数字 + 字 + 关键词” | 八只超低、六只全频 |
| 一对、一双 | 2只 |
| 两对 | 4只 |
| 三对 | 6只 |
| 串联、串接、并联串联、并连串接 | 并联串接 |

新增同音字时优先加到 `normalizeSpeakerVoiceText`，不要在多个解析正则里重复写。

## 意图模式

### 创建或追加

这些词表示继续添加新卡片：再加、再要、再来、还要、另外、还有、补一、补个、补上、追加、多一组、新建一组。

示例：

- “再加一对有缘全频” => 追加 2 只有源全频
- “然后再要3只并联的206M” => 追加 3 只 206M，并联串接 2 只/通道
- “还有两对DO118S” => 追加 4 只 DO118S
- “我要六只全瓶，八字超低” => 6 只全频 + 8 只超低
- 先说“我要8只全频，4只超低”，再说“全频要206M，超低型号是DO118S” => 给已有卡片补型号

### 修改已有

这些词表示改最近或已有卡片：刚刚、刚才、上一组、前面那个、前一组、这一组、这个、改成、不是、数量改为、换成。

示例：

- “刚刚的改成4只206M”
- “上一组换成DO115”
- “这个不是超低，是全频”

### 撤销

这些词表示回退一步：撤销、撤回、回退、上一步、取消刚刚、取消刚才、删掉刚刚、不要刚刚。

优先撤销最近一次语音对反推卡片的修改；没有语音快照时再尝试系统 `Store.undo()`。

### 并联串接

用户说“串联”时按系统正确术语纠正为“并联串接”。如果用户没有说每通道几只，默认 2 只/通道。

示例：

- “全频串联2只” => 全频并联串接 2 只/通道预览
- “206M要串接” => 识别为并联串接，缺数量时继续追问

### 系统设备忽略

功放、放大器、后级、DSP、处理器、调音台、数字台、交换机、Dante 不由语音直接指定。识别到这些词时提示：功放、DSP、调音台先保持智能选配，请继续告诉我音响数量、类型或型号。

## 槽位约定

```js
{
  intent: 'create_speaker_groups',
  mode: 'append' | 'modify' | 'replace',
  groups: [{
    role: 'fullrange' | 'sub',
    active: true | false,
    count: 2,
    templateQuery: '206M',
    templateName: '206M',
    connectionMode: 'parallel',
    unitsPerChannel: 2
  }]
}
```

有源音响 `active: true`，不参与功放反推，但会占用 DSP/调音台输出通道。无源音响 `active: false`。

## 追问文案

信息不全时不要结束语音会话，要鼓励用户继续说：

- 只有数量：收到，先记下 6 只。还差类型或型号，请继续说。
- 只有类型：好，听到全频了，还需要几只呢？
- 没听清：这句有点糊，再说一次就好。可以说：我要8只全频，4只超低。

## 新增规则检查清单

1. 新增同音字先放入归一化词库。
2. 新增意图先确认优先级，避免误吞“撤销”“刚刚的”等控制词。
3. 至少补一条 `SP.parseSpeakerVoiceCommand` 测试。
4. 如果会改变反推卡片，补一条 `SP.applySpeakerVoiceCommand` 应用测试。
5. 保持功放、DSP、调音台不被语音直接创建或选型。
